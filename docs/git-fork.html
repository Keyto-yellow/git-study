Githubのフォーク(Fork)機能について

  1. たとえばこういうときに使える！！
    • 他人が公開しているプロジェクトに参加したい！！
    • こういう書き方もできるんじゃないかなー？
  3. ・・・ってときに活躍する機能！！

実際に

  • 参加するプロジェクトを見つける
  • 自分のページにコピーされる
  • ローカルにcloneする
    • >> git clone [リポジトリ]
  • 自分用のブランチを作成する
    • masterのブランチはそのままの状態で保管しておいて、新しくブランチを作ってそこで編集する！！
    • >> git checkout -b [好きなブランチ名]
  • あとは編集してプッシュする
  • 「Compare & pull request」をクリックする
    • フォークしてきた自分のプロジェクトにプルリクエストのボタンが出てくる！！
  • コメントを添えてプルリクエストを送信
